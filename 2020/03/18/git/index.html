Git的使用

1.什么是Git
    一种源代码管理工具（版本控制工具），方便对代码的管理

2.初始化Git仓库
    在放置代码的文件夹下，
    右键“git bash here”,输入命令'git init',
    所有的文件放置在‘.git’文件夹下

3.配置身份信息
    声明当前的用户，“git config --global ”
    每次备份会将用户名和邮箱存储起来

    1.声明用户
        'git config --global user.name "用户名"'

    2.声明邮箱
        'git config --global user.mail "邮箱名"'
        邮箱不要求为现实邮箱，只要符合邮箱格式

4.将文件放置到Git
    自动将文件备份到隐藏文件夹‘.git’

    1.将代码添加到‘.git’仓库中（门口）
        'git add "文件路径名"'

    2.代码提交到‘.git’仓库中（房间）
        'git commit -m “对文件的说明”'
            -m message:必须要有文件说明

5.工作区 暂存区 版本库
    工作区：正在使用的文件夹
    暂存区：第一次添加的文件
    版本库：提交的源代码版本

6.查看当前状态
    'git status'
    文件是否放入版本库

7.批处理文件
    git add ./     :当前文件夹的所有修改过的内容放入暂存区
    git commit --all -m ""    :将所有的文件放入版本库

8.查看提交记录（日志）
    'git log' 显示提交的用户和时间
    简化提交日志
    'git log --oneline' :将日志简化为一条记录

9.恢复代码
    'git reset --hard Head~0'    ：回退到上次代码提交时的状态

10.根据版本号来进行回退
    'git reset --hard "版本号"'    ：将代码回退到某一次的提交时的状态

11.查看版本切换日志
    'git reflog'

12.创建分支，合并分支
    master:主分支

    1.创建分支
        'git branch "分支名"' 创建分支
        与master分支内容一样

    2.切换分支
        'git checkout "分支名"'    

    3.合并分支
        'git merge "分支名"',当前分支与指定分支的合并

    4.查看分支
        'git branch'

13.手动合并分支
    合并后删除不需要的文件，再次将其进行提交

14.将代码提交到Github服务器上
    'git push "地址" master'
    exp:git push https://github.com/print3306/test007.git master

15.将代码从Github上下载
    'git pull "地址" master'
    exp:git pull https://github.com/print3306/test007.git master

    注意：在下载之前必须初始化一个仓库，pull会对数据进行合并

16.对数据的clone
    'git clone "地址"'
    远程仓库数据的复制，如果多次执行会覆盖本地内容

17.通过SSH来上传代码
    公钥和私钥：通过两者之间的联系验证身份
    生成公钥和私钥
        ssh-keygen -t rsa -C "邮箱"
        生成的公钥添加到Github的SSH上

18.本地和远程文件的版本不一样
    先将服务器上的数据下载下来，在本地修改，解决冲突后，再将最新版上传至服务器

19.简化push和pull
    增加远程变量，在当前的项目目录有效，变量代替地址
    git remote add "变量名" "地址"

    上传：git push "变量名" -u master
        本地当前分支和远程指定分支进行相关联,
        下一次只需要写上git push进行上传

    下载：git pull "变量名" -u master
